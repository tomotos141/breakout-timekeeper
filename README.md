# ブレイクアウトルーム タイムキーパー

Zoomブレイクアウトルームでのグループディスカッション用タイムキーパーアプリ

## 目的

**タイムキーパーの負担を減らして、参加者が意見を出すことに集中できるようにする**

### 達成目標

| 目標 | 実現方法 |
|------|----------|
| ファシリテーター不要で進行できる | セクション名・残り時間・進捗図を常時表示 |
| 時間オーバーを防げる | 自動進行＋30秒/10秒前の音声アラート |
| 参加者全員が平等に発言できる | 3人×同一構成（21分ずつ）で進行 |
| 他の作業をしながら確認できる | PiPで別タブを見ながらタイマーを確認 |

## デモ

https://tomotos141.github.io/breakout-timekeeper/

## 機能

- **自動進行タイマー**: 設定時間が終わると自動で次のセクションへ
- **ピクチャーインピクチャー**: フローティングウィンドウで他のタブを見ながら確認・操作可能
- **進捗表示**: パンくずリストで全体の進行状況を可視化
- **音声アラート**: 30秒前・10秒前・終了時にビープ音
- **準備フェーズ ON/OFF**: 2回目以降のMTGでは①をスキップ可能
- **参加者名入力**: 名前を入力すると「〇〇さんの発表」と表示
- **余り時間の累計**: 早く終わった分を「貯金」として表示
- **任意の地点から再開**: スケジュールをクリックでジャンプ可能

## セッション構成

| セクション | 時間 |
|-----------|------|
| ① タイムキーパー・発表順の決定 | 2分 |
| ② 会社での役割 | 3分 |
| ③ 相談事項の共有 | 5分 |
| ④-1 チームメイトからのコメント | 5分 |
| ④-2 チームメイトからのコメント | 5分 |
| ⑤ 学び・気づき/babystepの共有 | 3分 |

※ ②〜⑤を3人分繰り返し（1人21分 × 3人 + 準備2分 = 計65分）

## 使い方

1. [デモページ](https://tomotos141.github.io/breakout-timekeeper/)を開く
2. Zoomで画面共有、またはミニ表示ボタンでフローティングウィンドウを開く
3. スタートボタンを押して開始

## ローカルで実行

```bash
# クローン
git clone https://github.com/tomotos141/breakout-timekeeper.git

# ブラウザで開く
open index.html  # macOS
start index.html # Windows
```

ビルド不要。ブラウザで直接開くだけで動作します。

## 動作環境

- Chrome 116以降（ピクチャーインピクチャー機能使用時）
- その他のブラウザでも基本機能は動作
